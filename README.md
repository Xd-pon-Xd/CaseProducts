# CaseProducts — Телеграм-бот для списка покупок

Бот принимает рецепт в любом виде — текст, ссылка на сайт или фото — и составляет итоговый список продуктов для магазина.

## Быстрый старт

### 1. Установить зависимости

```bash
C:\путь\к\venv\Scripts\python.exe -m pip install aiogram openai aiohttp beautifulsoup4 python-dotenv Pillow
```

### 2. Создать `.env`

Скопировать `.env.example` → `.env` и вставить ключи:

```
TELEGRAM_TOKEN=токен_от_BotFather
OPENROUTER_API_KEY=ключ_от_провайдера
```

> `.env` уже в `.gitignore` — в репозиторий не попадёт.

### 3. Запустить

```bash
C:\путь\к\venv\Scripts\python.exe tg.py
```

## Возможности

| Что отправить | Что получить |
|---|---|
| Текст рецепта | Список ингредиентов |
| Ссылка на кулинарный сайт | ИИ сам читает страницу и извлекает ингредиенты |
| Фото / скриншот рецепта | Vision-модель распознаёт список |
| Несколько рецептов подряд | Суммированная корзина (1 кг + 500 г = 1.5 кг) |

### Что умеет парсер
- Приводит названия к единой форме: «яйца» → «яйцо»
- Убирает пояснения: «масло для смазки» → «масло сливочное»
- При альтернативах («сливочное или растительное») добавляет оба варианта
- Угадывает количество при «щепотка», «по вкусу», «немного»
- Переводит названия с английского на русский
- Суммирует одинаковые ингредиенты из разных рецептов с переводом единиц

## Структура проекта

```
tg.py            — Telegram-бот (точка входа)
LLM_Parser.py    — LLM-парсер: текст, ссылки, фото
database.py      — SQLite: пользователи, рецепты, ингредиенты
.env             — секреты (не в git)
.env.example     — шаблон для .env
bot_errors.log   — лог ошибок (создаётся автоматически)
```

## Конфигурация модели

Модель и провайдер меняются в одном месте — `LLM_Parser.py`:

```python
client = AsyncOpenAI(
    base_url="https://polza.ai/api/v1",  # или https://openrouter.ai/api/v1
    api_key=api_key,
)
MODEL = "openai/gpt-5.5"  # любая модель провайдера
```

Для работы ссылок нужен провайдер с поддержкой **web-плагина** (OpenRouter).

## База данных

SQLite-файл `pantry.db` создаётся автоматически рядом с `tg.py`.

| Таблица | Что хранит |
|---|---|
| `users` | user_id, username, first_name |
| `ingredients` | ингредиенты с привязкой к рецепту и пользователю |
